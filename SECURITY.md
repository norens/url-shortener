# Security Policy

## Supported Versions

Only the latest version deployed from the `main` branch is supported with security updates.

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email **nazarfedyshyn@gmail.com** with:

- A description of the vulnerability.
- Steps to reproduce or a proof of concept.
- The potential impact.

## What Counts as a Security Issue

- Authentication or authorization bypass
- Data leaks or exposure of user data
- SQL injection, XSS, or other injection attacks
- CSRF or session-related vulnerabilities
- Secrets or credentials exposed in code

## Response Timeline

This is a solo open-source project maintained on a best-effort basis. I will do my best to:

- Acknowledge your report within **72 hours**.
- Provide an initial assessment within **1 week**.
- Ship a fix as soon as reasonably possible.

Thank you for helping keep Qurl safe.
